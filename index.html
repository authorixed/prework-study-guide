HTML

  • The head element contains information about the webpage.
    • Use < h1 > to < h6 > for size.
    • User wont see information in the head or meta elements.
    • Head is a container for metadata.
  • The style element defines the CSS styles used.
  • The base element defines the base URL for a webpage.
  • The link element connects an external source to the HTML doc.
  • The meta element defines metadata such as the char set, desc, keywords, ect.
  • The script element embeds client-side scripts in the HTML doc.
  • The title element changes the literal title of the tab.
  • The img element contains information about the image that are displayed.
    • The src attribute defines the location or source of the image file.
    • The alt attribute contains text that describes the image.
  • The br element creates a break between content.
  • The a element (or anchor) creates links to webpages.
    • The href attribute points tot he URL for the link.

CSS

  • A margin indicates the space around the element.
  • A padding indicates how much space we want around the content inside the element.
  • Must link the css style sheet in the html file in order for it to be added.
  • Order from inside out: content, padding, border, margin.
  • Can add CSS rules for two selectors by using a comma. example:
    • header,
      footer;
  • display: block; to make an element a block.
  • Static position: Normal flow, default position.
  • Relative position: Normal flow but is offset based on provided value.
  • Absolite position: element removed from normal flow, final position is relative to positioned parent.
  • FIxed position: element removed from normal flow, relativce to viewport and determined by values.
  • The * selector means UNIVERSAL selector. Anything in this will apply to everything. Usually done to reset to defaults for full control.
    • usually used by adding margin:0; and padding:0;
    • will NOToverride defined css elements, but will override html.
  • display: inline; allows elements to show to the left and right to each other on the same line
  • header: fixed; makes the header follow as you scroll, it'll stay at the top of your page constantly.
  • CSS comment out is /* */
  • How to make a site conform to different screen sizes!
      • @media screen and (max-width: 992px) {
    • This element says for all screen sizes up to 992px. Then add any elements!
    • Make sure you go from BIGGESTto SMALLESTsize!
  • @media print { sets the styles ONLY for printed content.
  • flex-wrap: wrap; Very important code! Does the math in the background to make the elements on the screen fit!
    • Will do its best to make the elements look good, it "flexes" to the screen! It will eventually wrap the elements under if they don't fit.
  • flex-shrink: sets how small an element can get if there isn't enough space
  • flex-grow: sets how big an element can get if there is space for it to take
  • flex-basis: sets the initial size of a content box unless specified otherwise!
      • Examples:
      flex-basis: auto;
      flex-basis: 0;
      flex-basis: 200px;
  • New element! justify-content: ;
    • Has TONS of options, very customizable and able to change the elements easily.
  • Using the > tag, (ie: article div > * ) means it has to be DIRECTLY inside that div for the style to apply.
  • li:nth-child(even) or (odd). dynamic selector? Reads I want to apply the style to list items to even or odd. Can substitude even/odd for number.
  • Targeting selectors are cool! ::(whatever) help target specific elements.
    • You can actually target attibutes through their href!
    • ex: a[href$='.css']
    • This code targets all files that end with .css on the page.
  • Brackets [] are nice to target very specific elements as well.
  • You can create your own variables in css by using :root {
      • create the variables by using '--' before it
    • ex: --dark: whatever color
    • helpful to change lots of things at once instead of having to indivitually change things
    • making a --border-radius: 50px can be used as one or multiple of the border radius px size.
      • ex:
        • var(border-radius) var(border-radius) 0 0
        this code means the left and right will be 50px, the top and bottom will be 0px.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • pwd shows what directory you're currently in.
  • ls command lists files in the directory.
  • cd.. moves you back one directory.
  • mkdir creates a directory.
  • start command opens that file. ex: start first-day.txt
  • How to NAVIGATE and CHECK specific branch
    • git status (checks current branch)
    • git checkout main (navigates to main branch)
    • git checkout whatever/feature (navigates to existing feature)
    • To make a NEW feature, add -b after checkout
  • How to UPDATE and MERGE branch
    • git add -A (adds all of the changed files to staging area)
    • git commit -m "insert update message here" (committing changes to repository)
    • git pull origin main (pulls updated branch before opening pull request)
    • git push origin feature/add-whatever (pushes all changes)
    • git checkout main (updates local main branch)
    • git pull origin main (pulls remote main branch into local main branch)
